# MASTER.md — Design System
_Generated by ui-pipeline on 2026-04-23. Stack: nextjs-shadcn. Source: Direct description + existing codebase inheritance._

## Product
- **Name**: 엑시트몰 (Exit Mall)
- **Type**: Closed B2B Marketplace (deposit-based) + Admin Backoffice (hybrid)
- **Industry**: 예치금 기반 폐쇄몰 / Wholesale e-commerce
- **Target user**:
  - Ordering side — 승인된 B2B 구매자 (예치금 보유)
  - Admin side — 운영팀 (주문/상품/고객/입금 관리)
- **Style direction**: modern, minimalist, professional, corporate/B2B trust, data-dense, restrained palette
- **Key features**:
  - 예치금(deposit) 기반 결제 (현금 잔액 차감)
  - 관리자 승인 가입 플로우 (pending → active)
  - 실시간 주문 토스트 (Supabase Realtime)
  - 상태 전이 기반 주문 관리 (placed → confirmed → shipped → delivered)
  - Storage 업로드 상품 이미지
  - 잔액 부족 임계치 알림

## Typography

- **Body font**: **Source Sans 3** — 휴머니스트 sans, 낮은 인지 부하

- **Mono font (숫자/코드)**: **JetBrains Mono** — 금액·송장번호·주문번호 탭ular

- **Scale**:
  - h1: 30px / 2.25rem (페이지 타이틀)
  - h2: 24px / 1.5rem (섹션)
  - h3: 20px / 1.25rem (카드 타이틀)
  - h4: 16px / 1rem 600 (subsection)
  - body: 14px / 0.875rem (기본 — 데이터 밀도 우선, 대시보드 관례)

  - small: 12px / 0.75rem (메타, 배지)

- **Weight**: heading = 600 (semi-bold, 너무 무겁지 않게), body = 400, emphasis = 500
- **Line-height**: heading 1.2, body 1.6 (1.5-1.75 범위)
- **Line-length**: 본문 65-75자 (max-w-prose ≈ 65ch)

## Spacing

- Scale: 4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64 / 96
- 카드 내부 padding: 24px (p-6)
- 페이지 컨테이너 padding: 24px mobile / 32px desktop
- 섹션 간격: 48px (gap-12)

## Border Radius
- Default: **6px** (rounded-md) — shadcn 기본
- Card: **8px** (rounded-lg)
- Button: **6px** (rounded-md)
- Badge / Pill: **9999px** (rounded-full, 상태 배지용)
- Input: **6px**
- _Swiss-style 영향: 과도한 둥글림 금지 (radius ≤ 10px)_

## Shadows
_Minimalist 원칙: 그림자는 최소화. 깊이는 border + 배경 contrast로_
- Card (rest):  `none` (border-only)
- Card (hover): `0 1px 2px 0 rgba(15, 23, 42, 0.04)` (아주 미묘)
- Modal/Popover: `0 10px 24px -6px rgba(15, 23, 42, 0.08), 0 4px 8px -2px rgba(15, 23, 42, 0.04)`
- Focus ring: `0 0 0 2px #0369A1` (CTA 컬러, 2px 링)

## Motion (Dial: MOTION=3 — 절제)
- Duration short: **150ms** (hover, focus)
- Duration medium: **200ms** (토글, 탭 전환)
- Duration long: **300ms** (모달, 드로어)
- Easing: `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo, UI 표준)
- Reduced-motion: `prefers-reduced-motion: reduce` 시 모든 transition 0ms
- **사용 허용**: 버튼 hover, 포커스 링 fade-in, 토스트 slide-in, 모달 fade+scale, 스켈레톤 pulse
- **사용 금지**: 페이지 로드 스태거 애니, 장식용 반복 애니, parallax, 스크롤 하이재킹

## Layout (Dial: VARIANCE=1 — 표준, DENSITY=8 — 고밀도)
- Grid: 12열 (Swiss Modernism 2.0 원칙)
- Max container width: **1280px** (max-w-7xl)
- Sidebar (admin): 240px fixed (lg+) / drawer (mobile)
- Content area: grid 12 cols, gap 24px
- 테이블: density-high — row height 44px, padding 8px 12px, stripe 없음, hover only
- 카드 grid: 3 cols (lg) / 2 cols (md) / 1 col (sm)

## Anti-patterns (사용 금지)
- 장식용 무한 애니메이션 (스피너/스켈레톤만 허용)
- linear easing
- alt 없는 img, label 없는 input
- 시각적으로만 에러 표시 (aria-live 또는 role="alert" 필수)
- 4.5:1 미만 텍스트 대비
- 순수 검정 `#000000` 텍스트 (→ slate-950 사용)
- Arial / Roboto / Times 기본 폰트
- 카드 안에 카드 중첩 (depth 2 초과 금지)
- prefers-reduced-motion 미적용
- 하드코딩된 컬러 (CSS 변수/Tailwind 토큰만 사용)
- 금액에 가변 폰트 (tabular-nums / mono 필수)
- 작은 터치 타겟 (<44x44px) — 특히 테이블 액션 버튼
- 모달 중첩 (confirm on top of form 금지, 인라인 confirm 사용)
- 이모지 아이콘 (lucide-react SVG 사용)
- 한국어 텍스트에 영문 letter-spacing 적용
- toast 외 상태 변경 시각적 피드백 누락

## Stack-specific rules (nextjs-shadcn)
- Server Components 우선, client boundary는 Form/Realtime/interactive만
- `useFormState` / `useFormStatus` 활용한 Server Action form
- shadcn 컴포넌트는 CVA variant 패턴 유지, 직접 수정보다 wrapper 권장
- `cn()` 유틸 통한 클래스 병합, `clsx` + `tailwind-merge` 순
- Image는 `next/image` + `sizes` 지정 (상품 썸네일)
- Route Handler / Server Action에서 Zod 검증 먼저, 실패 시 문맥 있는 에러 반환
- 로그인 세션은 `createServerClient` (Supabase SSR 쿠키 전파)

## Data Display Conventions (데이터 밀도 8)
- 금액: `toLocaleString('ko-KR')` + `₩` prefix, 우측 정렬, JetBrains Mono
- 날짜: `YYYY-MM-DD HH:mm` 기본, 목록에서는 `MM-DD`
- 상태 배지: pill shape, 4색만 (success/warning/danger/info)
- Empty state: 아이콘 + 1줄 설명 + 단일 CTA (overengineered 일러스트 금지)
- Loading: 스켈레톤 우선 (동일 레이아웃), 스피너는 버튼 내부만
- Error state: inline (form field) + page-level (destructive variant)

## Screens to generate (16개, 우선순위 순)
**Tier 1 — 핵심 플로우 (먼저 Stitch)**

9. `/orders` — 내 주문 내역 (상태 필터 + 취소 CTA)

**Tier 3 — 관리자 세부**
11. `/admin/approvals` — 가입 승인 (pending 목록 + approve/reject)
12. `/admin/deposits` — 입금 확인 (이체 대조 + 승인)
13. `/admin/products` — 상품 CRUD (드로어/모달 + 이미지 업로드)
14. `/admin/users` — 사용자 관리 (잔액 조정 + 상태 변경 + 임계치)

16. `/admin/settings` — 계좌 정보 (단일 폼)
